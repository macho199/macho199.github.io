모바일 태그 필터 헤더 구현계획

태그 목록과 게시글 수가 멀리 떨어져 보이는 모바일 홈을 하나의 정보 영역으로 정리합니다.

작성일2026-07-19 작성자Luna 범위홈 화면 · 720px 이하

한 줄 요약

모바일에서 첫 행은 태그 필터 · N posts, 둘째 행은 전체 폭 태그 목록으로 만들고 PC 레이아웃은 그대로 유지합니다.

01배경 · 왜

현재 모바일에서는 여러 줄 태그 목록 아래에 결과 수가 따로 놓여 두 정보가 하나의 필터 영역으로 읽히지 않습니다.

문제

태그가 두 줄 이상으로 개행되면서 결과 수가 아래쪽으로 밀립니다. 필터 기준과 결과 수 사이의 시각적 거리가 커져 목록 시작점도 불필요하게 내려갑니다.

승인된 A안은 헤더와 태그 목록의 역할을 분리합니다. 짧은 라벨과 동적 결과 수를 한 행에 묶고, 실제 태그 버튼은 다음 행에서 전체 폭을 사용합니다.

02접근 · 설계 결정

필터 데이터 로직은 건드리지 않고 표현 경계인 PostFilterBar와 홈 CSS만 조정합니다.

결정선택 이유
가시 라벨 추가태그 필터가 결과 수의 의미를 바로 설명합니다. 접근성 이름은 기존 toolbar와 group을 유지하고 새 라벨은 중복 낭독되지 않게 처리합니다.
모바일 grid 재배치첫 행은 라벨·결과 수, 둘째 행은 태그 그룹으로 구성합니다. 721px 이상 기본 grid는 변경하지 않습니다.
숨김·더보기 제외태그 수가 많지 않고 모든 선택지를 바로 보는 것이 검색보다 단순합니다.
가로 스크롤 제외모바일에서 필터 선택지를 놓치기 쉽고 기존의 자연 개행 동작과도 맞지 않습니다.

03확정 vs 추정

현재 코드와 승인된 설계에서 확인된 내용, 브라우저 검증으로 확정할 내용을 분리합니다.

구분내용
확정기존 toolbar는 PC에서 태그 왼쪽·결과 수 오른쪽의 2열 grid이며, 모바일에서 1열로 바뀝니다.
확정모바일 간격은 헤더→태그 8px, 태그→게시글 목록 12px입니다. 모든 태그를 숨김 없이 표시합니다.
확정단일 선택, aria-pressed, live status, 1 post / N posts 규칙은 유지합니다.
추정390px에서 라벨과 결과 수가 겹치지 않을 것으로 예상합니다. 짧은 고정 라벨과 현재 결과 수 형식에 근거하며 Chromium·WebKit 좌표 검증으로 확정합니다.

05완료 기준

아래 항목을 코드 계약과 실제 브라우저에서 모두 확인하면 구현 완료입니다.

수용 기준

  • 720px 이하 첫 행에 태그 필터와 동적 N post/posts가 함께 보입니다.
  • 모든 태그가 둘째 행 전체 폭에서 자연스럽게 개행되고 가로 스크롤이 없습니다.
  • 헤더→태그는 8px, 태그→게시글 목록은 12px입니다.
  • 721px 이상 PC 배치·16px 열 간격·20px 하단 여백은 변하지 않습니다.
  • 태그가 없으면 라벨과 버튼 그룹 없이 결과 수만 오른쪽에 표시됩니다.
  • 기존 필터 동작과 접근성 상태가 유지되며 Chromium·WebKit 검증을 통과합니다.

06범위 밖 · 리스크

이번 작업은 모바일 필터 정보 구조와 간격 수정에만 한정합니다.

이번엔 하지 않음

태그 검색·더보기·가로 스크롤·URL 동기화·history·localStorage·필터 데이터 로직·신규 패키지는 변경하지 않습니다. push·PR·병합·배포도 구현 완료 후 별도 승인 범위입니다.

리스크 WebKit의 글꼴 폭과 줄바꿈 결과는 Chromium과 다를 수 있습니다. 정확한 줄 수를 고정하지 않고, 둘 이상으로 개행되는지와 가로 넘침이 없는지를 수용 기준으로 사용합니다.
