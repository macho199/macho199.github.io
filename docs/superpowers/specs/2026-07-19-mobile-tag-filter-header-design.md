# Developer Blog 모바일 태그 필터 정보 영역 설계

## 목적

모바일 홈에서 여러 줄 태그와 게시글 수가 분리돼 보이는 문제를 해결한다. 태그 필터와 현재 결과 수를 작은 헤더로 묶어 정보 위계를 만들고, 태그가 늘어나도 전체 폭에서 안정적으로 개행되게 한다.

## 사용자 결과

- 모바일에서 `태그 필터`와 현재 게시글 수를 한눈에 확인할 수 있다.
- 모든 태그가 숨김·가로 스크롤 없이 전체 폭에서 자연스럽게 개행된다.
- 태그 영역과 게시글 목록 사이의 불필요한 세로 여백이 줄어든다.
- PC의 현재 태그·게시글 수 배치는 바뀌지 않는다.

## 현재 원인

- 720px 이하에서 `.post-filter-toolbar`는 1열 grid로 전환된다.
- 390px에서 9개 필터 버튼은 3줄이 되고, 게시글 수는 12px 아래의 별도 행에 오른쪽 정렬된다.
- 게시글 수 아래에는 toolbar의 20px 하단 여백이 추가된다.
- Chromium과 WebKit의 좌표가 같으므로 브라우저별 렌더링 문제가 아니라 모바일 레이아웃 문제다.

## 범위

### 포함

- `PostFilterBar`의 모바일용 가시 라벨 `태그 필터`
- 모바일에서 라벨과 동적 게시글 수를 같은 헤더 행에 배치
- 태그 버튼 그룹을 헤더 아래 전체 폭에 배치
- 모바일 헤더→태그 8px, 태그→게시글 목록 12px 간격
- 기존 필터·결과 수·접근성 동작 유지
- 태그가 없는 경우의 단독 게시글 수 배치
- 390·430·720·721px 반응형 검증

### 제외

- 태그 가로 스크롤
- 태그 더보기·접기
- 태그 검색 입력창
- URL·history·localStorage 상태
- 태그 수집·필터 데이터 로직
- 게시글 카드와 PC 레이아웃 재설계

## 컴포넌트 경계

### `PostFilterBar`

- 기존 props와 필터 버튼 생성 로직을 유지한다.
- 태그가 있을 때만 가시 라벨을 렌더링한다.
- DOM 순서는 라벨 → 태그 버튼 그룹 → 결과 수를 유지한다.
- 결과 수는 기존 `role="status"`와 `aria-live="polite"`를 유지한다.

### `PostList`와 필터 모듈

- 선택 상태와 필터 결과 계산을 그대로 유지한다.
- 이 작업에서 데이터 흐름과 함수 계약을 변경하지 않는다.

## 화면 규칙

### PC: 721px 이상

- toolbar는 기존 `minmax(0, 1fr) auto` 2열을 유지한다.
- 태그 버튼 그룹은 왼쪽, 게시글 수는 오른쪽 위에 둔다.
- 모바일용 `태그 필터` 라벨은 표시하지 않는다.
- 기존 16px 열 간격과 20px toolbar 하단 여백을 유지한다.

### 모바일: 720px 이하

- toolbar는 `minmax(0, 1fr) auto` 2열, 2행 grid로 구성한다.
- 첫 행은 왼쪽 `태그 필터`, 오른쪽 `N post/posts`다.
- 둘째 행의 태그 버튼 그룹은 두 열 전체를 사용한다.
- 첫째 행과 태그 사이 row gap은 8px다.
- toolbar 하단 여백은 12px다.
- 태그 버튼의 기존 가로 16px·세로 8px gap은 유지한다.
- 390px과 430px에서 태그가 3줄이 되어도 페이지 가로 overflow가 없어야 한다.

## 시각 표현

- `태그 필터`는 본문 글꼴, 12px, 500 굵기, 보조 전경색을 사용한다.
- 게시글 수는 기존 고정폭 글꼴·색상·복수형 규칙을 유지한다.
- 활성 태그의 연초록 밑줄, 굵기, `#` 접두사 표현을 유지한다.
- 새 배경·테두리·아이콘은 추가하지 않는다.

## 접근성

- toolbar의 `aria-label="글 필터"`와 버튼 그룹의 `aria-label="태그 필터"`를 유지한다.
- 가시 라벨은 태그가 있을 때만 렌더링하되 기존 접근 이름을 대체하지 않는다.
- 태그 버튼은 native `button`과 `aria-pressed`를 유지한다.
- 결과 수 live status는 DOM에서 버튼 그룹 뒤에 두어 선택 결과를 기존 순서대로 알린다.
- CSS grid 배치가 keyboard focus 순서를 바꾸지 않게 한다.

## 빈 상태와 방어 규칙

- 태그가 없으면 가시 라벨과 버튼 그룹을 렌더링하지 않는다.
- 게시글 수는 현재 전체 결과 수를 단독으로 오른쪽 정렬한다. 게시글 자체가 없을 때만 `0 posts`다.
- 존재하지 않는 선택값의 전체 목록 fallback은 기존 필터 모듈에 맡긴다.
- 태그 개수나 문자열 길이를 기준으로 숨기거나 자르지 않는다.

## 검증

### 자동 검증

- `PostFilterBar`가 태그가 있을 때 가시 라벨을 렌더링하는 계약을 추가한다.
- 모바일 CSS가 2열·2행 배치, 8px row gap, 12px 하단 여백을 갖는지 확인한다.
- 태그 버튼 그룹이 모바일에서 두 열 전체를 사용하고 결과 수가 첫 행 오른쪽에 놓이는지 확인한다.
- 기존 `aria-pressed`, live status, PC grid 계약을 유지한다.
- 전체 테스트, typecheck, Gatsby production build, 정적 verifier를 통과한다.

### 브라우저 검증

- 390·430px에서 헤더 한 줄, 태그 3줄, 가로 overflow 0을 확인한다.
- 720px에서 모바일 헤더와 태그 개행을 확인한다.
- 721px에서 기존 PC 2열 toolbar로 전환되는지 확인한다.
- `전체`, `GraphQL`, `SEO`를 선택해 결과 수·목록·`aria-pressed`가 함께 갱신되는지 확인한다.
- Chromium과 WebKit에서 같은 배치와 간격을 확인한다.

## 완료 조건

- 모바일에서 `태그 필터`와 게시글 수가 같은 헤더 행에 표시된다.
- 모든 태그가 전체 폭에서 자연스럽게 개행된다.
- 모바일 헤더→태그 8px, 태그→게시글 목록 12px가 적용된다.
- PC 레이아웃과 필터·접근성 동작이 유지된다.
- 신규 런타임 패키지를 추가하지 않는다.
