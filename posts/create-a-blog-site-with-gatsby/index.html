January 3, 2024

요즘 세상에 나만의 블로그를 만드는 건 누워서 떡먹기입니다. 대기업에서 만든 블로그 플랫폼인 네이버 블로그, 티스토리, 구글 블로거를 이용하면 몇 번의 클릭만으로 나만의 블로그를 만들 수 있죠. 그것도 아주 예쁘게!

하지만 무릇 개발자라면 블로그 정도는 내 손으로 처음부터 끝까지 만들어봐야 되는게 아닌가요? 내 취향의 디자인으로 만든 블로그라니 벌써부터 심장이 두근거립니다. 나만의 서버를 만들어 블로그 사이트를 운영할 수도 있지만 비용 및 운영 측면에서 더 나은 선택지가 있어요. 바로 Github Pages입니다.

Github Pages는 정적 웹 호스팅 서비스로 HTML, CSS, Javascript로 구현한 코드를 나의 Github 저장소에 push하면 웹사이트를 24시간 무중단과 함께 공짜로 운영할 수 있죠! Github Pages 소개페이지에 저장소를 생성하는 방법이 쉽게 설명되어 있어요. 이후 구현된 사이트를 저장소에 push할 때 다시 얘기해 보기로 하죠.

정적 사이트 생성기(SSG)

HTML, CSS, Javascript만으로도 블로그 사이트를 만들 수 있지만, Jekyll이나 이 포스트에서 얘기할 Gatsby를 사용한다면 자체 데이터 레이어를 통해 쉽게 페이지를 관리할 수 있고, 모듈을 재사용하는 방식으로 간소화된 코드로 개발 및 유지보수하기 용이합니다. 물론 그에 따른 러닝커브도 존재하지만요..

React 라이브러리로 webpack이나 요즘 핫한 Vite 같은 모듈번들러를 이용하면 손쉽게 DOM을 그리는(렌더링하는) javascript파일을 추출할 수 있습니다. 즉 javascript가 실행되어야지 브라우저에 화면이 노출된다는 겁니다.

이와 달리 Gatsby 같은 정적 사이트 생성기는 화면을 그리는(렌더링하는)것을 미리 번들링하여 html파일로 만들어지게 된다는 점입니다.

이렇게 다른 결과물의 차이로 빠른 성능과 SEO 최적화를 얻을 수 있습니다.

동적인 페이지로도 구현은 가능하지만 동적인 요소가 늘어나면 늘어날 수록 그만큼 Page가 생성되니, 용량도 커지고 빌드시간도 길어진다고 한답니다.

Gatsby

Gatsby를 간단히 말하면 React를 사용하여 정적 웹사이트를 만들 수 있게 도와주는 프레임워크입니다.. Gatsby의 주요 특징은 다음과 같습니다.

  1. 빠른 성능: Gatsby는 정적 사이트 생성(Static Site Generation)을 기반으로 하기 때문에 페이지 로딩 속도가 빠르고 우수하다.
  2. SEO 최적화: Gatsby는 검색 엔진 최적화(SEO)를 위한 필수적인 기능들을 기본적으로 제공한다. 사이트 구조, 메타데이터, 이미지 최적화 등을 간편하게 관리할 수 있어서 검색 엔진에서 사이트가 더 잘 노출되도록 할 수 있다.
  3. 안정성과 보안: 정적 사이트는 서버 취약점에 대한 위험이 줄어들어 보안 측면에서 더 안전하며, 서버에 대한 부담이 줄어들어 사이트의 안정성이 향상될 수 있다.
  4. 모듈화와 확장성: Gatsby의 플러그인 시스템을 활용하면 다양한 기능을 간단히 추가하고 사용할 수 있어, 이를 통해 플로그에 필요한 다양한 모듈들을 쉽게 통합할 수 있어 유연성이 높아진다.
  5. 반응형 웹 디자인: Gatsby는 반응형 웹 디자인에 적합하며, 모바일 기기에서도 훌륭한 사용자 경험을 제공할 수 있다. 이는 블로그가 다양한 디바이스에서 잘 보이고 사용될 수 있다는 것을 의미한다.
  6. 커뮤니티 지원과 자료: Gatsby는 커뮤니티 기반으로 발전하고 있어서, 다양한 템플릿, 플러그인, 자료들을 공유하고 있다. 이로 인해 새로운 기능을 추가하거나 문제를 해결하는 데 도움을 받을 수 있다.

Gatsby 설치와 프로젝트 실행

Gatsby 설치 방법과 주요 사용법은 Gatsby 공식 홈페이지 Getting Started를 직접 해보면 어느정도 감이 잡히실 겁니다.

설치 버전
  • Node.js: v20.9.0
  • Gatsby: 5.13.0

이제 설치가 완료되었으니 Gatsby 사이트 개발서버를 실행해볼까요?

$ cd my-gatsby-site
$ npm start

Invalid plugin options for "gatsby-plugin-google-gtag":

gatsby-config.js 파일을 열어 plugins 배열에 추가되어 있는 gatsby-plugin-google-gtag 플러그인을 삭제하고 아래 부분에 객체로 기본설정을 해주고 다시 개발서버를 실행(npm start)해 보면, 정상적으로 실행되며 2개의 사이트 접속 주소가 나옵니다.

  • http://localhot:8000: Gatsby 사이트
  • http://localhot:8000/___graphql: GraphQL 데이터 레이어 인터페이스

GraphQL은 다음 장에서 보기로 하고 http://localhot:8000 Gatsby 개발사이트로 접속해 봅시다.

Github로 생성한 정적 사이트 배포하기

아직 만들어진건 없지만, Gatsby에서 기본으로 만들어준 Welcome사이트를 Github 나의 저장소에 배포해서 호스팅을 해볼까요?

우선 지금 실행되고 있는 개발서버를 종료해야 합니다. 터미널 창에서 ctrl + c를 하면 작업을 끝내실 거냐고(Y/N)으로 물어보는데 Y를 입력하고 엔터를 하셔도 되고 ctrl + c를 한번 더 하셔도 종료됩니다.

Gatsby 프로젝트 최상단 폴더로 이동하시면 .cache와 public 폴더가 생성되어 있을겁니다. 개발서버를 위해 생성된 폴더인데요. npm clean 명령어로 깔끔하게 삭제해주세요.

그 다음 드디어 배포버전의 정적 사이트를 생성해 봅시다.

$ npm run build

빌드가 완료되면 .cache 폴더와 public 폴더가 다시 생성되어 있을겁니다. public 폴더의 파일들이 바로 정적 사이트 입니다. 웹 브라우저로 해당 경로를 접속하면 아까 봤던 Welcome 사이트가 잘 나오는 것도 확인하실 수 있습니다. 예로 c:\에 프로젝트를 설치하셨다면 c:\my-gatsby-site\public\index.html 이런식으로 브라우저 주소창에 입력하시면 보일 겁니다.

이제 public 폴더 안에 파일을 그대로 복사해서 commit하고 push하면 됩니다.

Github 저장소에 정적 사이트 소스 push

Github Pages 소개란에 있는 것처럼 {github username}.github.io 패턴명으로 저장소를 생성하셨죠?

그럼 다음으로 생성한 저장소의 설정을 추가로 수정해줘야 될게 있습니다.

  1. 저장소 상단 Settings 클릭
  2. 왼쪽 메뉴에서 Pages 클릭
  3. Build and deployment 설정
    • Source: Deploy from a branch
    • Branch: main, /(root)

이렇게 추가로 설정해주면 끝납니다.

이제 생성한 저장소를 git clone를 통해 내 로컬로 복사하고 해당 폴더에 있는 파일은 모두 삭제하고 public 폴더를 복사한 후

$ git add
$ git commit -m "init"
$ git push -u origin main

이렇게 저장소를 commit하고 push 해준뒤에 브라우저에서 {github username}.github.io 접속하면 아까 개발할때 봤던 페이지가 동일하게 나오시는 것을 확인하실 수 있어요~

처음 저장소를 push하면 사이트가 활성화 되기 까지 시간이 오래 걸립니다.(10분 ~ 30분)

이번 포스트는 여기까지 하고, 다음 포스트에서 전반적으로 사이트를 개발하는 방법을 다뤄볼께요!
