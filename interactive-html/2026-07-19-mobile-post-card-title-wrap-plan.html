모바일 포스트 카드 제목 줄바꿈 수정

카드 제목만 전역 균형 줄바꿈에서 분리하고, 한국어 단어 경계를 보존하는 최소 수정 계획입니다.

작성일2026-07-19 작성자Luna 범위GitHub #38 · Redmine #30

한 줄 요약

home.css의 카드 제목 규칙과 회귀 테스트만 수정해 남는 폭을 사용하고 단어 내부 분리를 막습니다.

01배경 · 왜 지금 수정하는가

모바일 카드에는 345px 폭이 있지만 제목 첫 줄은 약 167~201px에서 끝납니다.

카드 제목이 전역 heading의 text-wrap: balance를 상속해 두 줄 길이를 비슷하게 맞추고 있습니다. 링크의 overflow-wrap: anywhere는 사이트 같은 한국어 단어 내부도 줄바꿈 후보로 허용합니다.

진단 결과

Chromium과 WebKit 모두 제목 박스 폭은 정상이고 가로 overflow도 없습니다. 레이아웃 폭이 아니라 카드 범위에 맞지 않는 텍스트 규칙이 원인입니다.

02접근 · 설계 결정

전역 규칙은 유지하고 홈 카드 선택자에서만 줄바꿈 방식을 덮어씁니다.

Before — 균형 배치와 임의 분리
text-wrap: balance첫 줄에 넓은 빈 공간
→
overflow-wrap: anywhere단어 내부 분리 허용
After — 폭 우선과 단어 보존
text-wrap: wrap사용 가능한 폭 우선
→
keep-all + break-word단어 우선, 긴 토큰 fallback
대안판단
카드 전체 구간 적용채택. 컴포넌트 의미가 breakpoint와 무관하고 넓은 화면의 한 줄 제목에는 변화가 없습니다.
모바일 media query만 적용보류. 720px 경계에서 같은 카드가 서로 다른 줄바꿈 규칙을 갖게 됩니다.
전역 heading 수정제외. 상세 포스트와 다른 화면까지 영향을 넓힙니다.

03확정된 사실과 구현 후 확인할 것

상태근거와 확인 항목
확정393px에서 제목 박스는 345px입니다. balance만 해제하면 첫 줄 사용 폭이 약 327~344px로 늘었습니다.
확정keep-all과 break-word 조합은 일반 단어 경계를 우선하고 긴 토큰 fallback을 남깁니다.
구현 후 검증720·721·1020·1440px에서 태그·요약·날짜 간격과 가로 overflow가 그대로인지 두 엔진에서 확인합니다.

05완료 기준

검증 기준

  • 모바일 제목이 사용 가능한 폭을 우선 사용합니다.
  • 한국어 단어 내부 분리와 페이지 가로 overflow가 없습니다.
  • 전체 테스트·타입 검사·빌드·정적 verifier가 통과합니다.
  • 상세 포스트, 전역 heading, 홈 필터와 콘텐츠 데이터는 변하지 않습니다.

06범위 밖 · 리스크

이번 작업에서 하지 않음

전역 heading·상세 포스트·카드 타이포그래피·콘텐츠·필터·배포는 변경하지 않습니다. push, PR, 이슈 종료도 별도 승인 전에는 수행하지 않습니다.

주요 리스크는 엔진별 글꼴 폭 차이입니다. 특정 줄 문자열을 고정하지 않고 제목 폭 사용률, 공백 경계, overflow와 요소 겹침으로 검증해 차이를 흡수합니다.
