#gatsby#github-pages#react#tailwindcss

개발자를 위한 블로그 구축

블로그 플랫폼은 네이버 블로그, 티스토리, 구글 블로거 등 다양한 선택지가 있습니다. 이러한 플랫폼들은 빠른 시작과 편리한 관리 기능을 제공하지만, 개발자에게는 제한적인 커스터마이징과 플랫폼 종속성이라는 한계가 존재합니다.

개발자가 자신만의 블로그를 구축하는 이유는 명확합니다. 완전한 제어권과 원하는 기능의 자유로운 구현이 가능하기 때문입니다.

물론 개인 서버를 구축하여 운영하는 방법도 있지만, 이는 상당한 비용과 지속적인 관리 부담을 수반합니다. 이러한 문제를 해결할 수 있는 효과적인 대안이 바로 Github Pages를 활용한 정적 사이트 호스팅입니다.

Github Pages의 주요 장점

Github Pages는 정적 웹 호스팅 서비스로, HTML, CSS, Javascript로 구현된 코드를 Github 저장소에 배포하면 24시간 안정적이고 무료로 웹사이트를 운영할 수 있습니다.

Github Pages는 개인 프로젝트와 오픈소스 프로젝트에 대해 완전히 무료로 제공됩니다.

Github Pages 소개페이지에서 저장소 생성 방법에 대한 자세한 설명을 확인할 수 있으며, 이는 사이트 배포 단계에서 참고하겠습니다.

정적 사이트 생성기(SSG) 이해하기

순수 HTML, CSS, Javascript만으로도 블로그 구축이 가능하지만, Jekyll이나 Gatsby 같은 정적 사이트 생성기를 활용하면 다음과 같은 이점을 얻을 수 있습니다:

  • 자체 데이터 레이어: 페이지와 콘텐츠 관리의 효율성 향상
  • 모듈 재사용: 컴포넌트 기반 개발로 코드 중복 최소화
  • 유지보수성: 구조화된 아키텍처를 통한 장기적 관리 용이성

물론 이러한 도구들은 학습 곡선이 존재하지만, 한 번 습득하면 다양한 프로젝트에서 활용할 수 있는 범용성을 제공합니다.

React 앱과 Gatsby의 차이점

React로 webpack이나 Vite 같은 번들러를 사용한 일반적인 Single Page Application(SPA)은 클라이언트 사이드에서 Javascript가 실행되어야 화면이 렌더링됩니다.

반면 Gatsby 같은 정적 사이트 생성기는 빌드 타임에 미리 렌더링을 완료하여 HTML 파일을 생성합니다.

이러한 접근 방식의 차이로 인해 다음과 같은 장점을 얻을 수 있습니다:

  • 성능 최적화: 초기 로딩 시간 단축
  • SEO 최적화: 검색 엔진 크롤러가 완성된 HTML을 읽을 수 있음

참고: 동적 기능 구현도 가능하지만, 동적 요소가 증가할수록 빌드 시간과 결과물 크기가 증가할 수 있습니다.

Gatsby 프레임워크 소개

Gatsby는 React 기반의 정적 사이트 생성 프레임워크로, 현대적인 웹 개발 환경에서 고성능 웹사이트를 구축할 수 있도록 설계되었습니다.

Gatsby의 핵심 장점

1. 성능 최적화 - 정적 사이트 생성 방식을 통해 뛰어난 페이지 로딩 속도를 제공합니다.

2. SEO 최적화 - 검색 엔진 최적화를 위한 필수 기능들이 내장되어 있습니다. 사이트 구조, 메타데이터, 이미지 최적화 등을 효율적으로 관리할 수 있습니다.

3. 보안과 안정성 - 정적 사이트의 특성상 서버 취약점에 대한 위험이 현저히 낮으며, 안정적인 서비스 운영이 가능합니다.

4. 확장성과 모듈화 - 풍부한 플러그인 생태계를 통해 필요한 기능을 쉽게 확장할 수 있습니다.

5. 반응형 디자인 - React 기반 아키텍처를 통해 유연한 반응형 디자인과 인터랙티브한 사용자 경험을 구현할 수 있습니다.

Gatsby 설치 및 프로젝트 생성

Gatsby 공식 홈페이지의 Getting Started 가이드를 참고하여 프로젝트를 시작하겠습니다.

환경 요구사항: Node.js v20.x

Gatsby CLI 설치

터미널에서 다음 명령어를 실행합니다:

Shell
$ npm install -g gatsby-cli

새 프로젝트 생성

설치된 Gatsby CLI를 사용하여 새 프로젝트를 생성합니다.

Shell
$ gatsby new

명령어 실행 후 나타나는 설정 옵션들을 다음과 같이 선택합니다:

Shell

? >> / my-gatsby-site

설정 옵션 설명

  • JavaScript vs TypeScript: TypeScript의 장점은 많지만, 빠른 시작을 위해 JavaScript를 선택했습니다.
  • Styling System: 일관된 디자인 시스템 구축을 위해 TailwindCSS를 선택했습니다.
  • 추가 플러그인: SEO 최적화 관련 플러그인들과 확장된 마크다운 기능인 MDX 플러그인을 포함했습니다.

참고: 현재 선택하지 않은 옵션들도 프로젝트 진행 중 언제든 추가할 수 있습니다.

설치 완료 후 개발 서버를 실행합니다:

Shell
$ cd my-gatsby-site && npm run develop

TailwindCSS 설정 오류 발생

알려진 이슈: 2025년 8월 기준 Gatsby 5.14.6 버전에서 TailwindCSS 관련 설정 오류가 발생할 수 있습니다. 다음 단계를 통해 해결하겠습니다.

오류 해결 과정

1단계: 패키지 버전 호환성 해결

MDX 관련 패키지 버전 충돌을 해결하기 위해 package.json 파일을 수정합니다:

JSON
{
  "dependencies": {
    "@mdx-js/react": "^2.3.0" // ^3.1.0에서 ^2.3.0으로 변경
  }
}

2단계: TailwindCSS PostCSS 플러그인 설치

Shell
$ npm install @tailwindcss/postcss

설치 중 보안 취약점 경고가 발생하는 경우:

Shell
$ npm audit fix

3단계: PostCSS 설정 수정

postcss.config.js 파일을 다음과 같이 수정합니다:

JavaScript
// postcss.config.js
module.exports = {
  plugins: {
    "@tailwindcss/postcss": {}, // 'tailwindcss': {} 에서 변경
    autoprefixer: {},
  },
};

4단계: TailwindCSS 스타일 import 방식 변경

src/styles/global.css 파일을 수정합니다:

CSS
/* src/styles/global.css */
/* 기존 방식 (주석 처리) */
/* @tailwind base;
@tailwind components;
@tailwind utilities; */

/* 새로운 방식 */
@import "tailwindcss";

수정 완료 후 개발 서버를 재실행합니다:

Shell
$ npm run develop

개발 서버 정상 실행 확인

사이트 확인 및 테스트

개발 서버가 정상적으로 실행되면 다음 주소들에 접근할 수 있습니다:

  • 메인 사이트: http://localhost:8000
  • GraphQL 인터페이스: http://localhost:8000/___graphql

GraphQL 데이터 레이어에 대한 자세한 내용은 이후 포스트에서 다루겠습니다. 현재는 메인 사이트 확인에 집중하겠습니다.

TailwindCSS 동작 확인

TailwindCSS가 정상적으로 적용되는지 확인하기 위해 기본 환영 문구의 스타일을 수정해보겠습니다.

text-red-500 클래스를 추가하여 제목을 빨간색으로 변경합니다:

JSX
// src/pages/index.js
<h1 style={headingStyles} className="text-red-500">
  Congratulations
</h1>

Github Pages 배포

로컬 개발이 완료되었으므로 Github Pages를 통해 사이트를 배포하겠습니다.

1단계: 개발 서버 종료

터미널에서 Ctrl + C를 눌러 개발 서버를 종료합니다.

2단계: 임시 파일 정리

개발 중 생성된 캐시와 임시 파일들을 정리합니다:

Shell
$ npm run clean

3단계: 프로덕션 빌드

배포용 정적 사이트를 생성합니다:

Shell
$ npm run build

빌드 완료 후 public 폴더가 생성되며, 이 폴더에 포함된 파일들이 실제 배포될 정적 사이트입니다.

확인 방법: 웹 브라우저에서 file:///[프로젝트 경로]/my-gatsby-site/public/index.html 경로로 접속하여 빌드된 사이트를 확인할 수 있습니다.

4단계: Github 저장소 설정

Github Pages 소개페이지를 참고하여 {github username}.github.io 형식으로 저장소를 생성한 후, 추가 설정을 진행합니다:

설정 절차:

  1. 저장소 상단의 Settings 메뉴 선택
  2. 좌측 메뉴에서 Pages 선택
  3. Build and deployment 설정:
    • Source: Deploy from a branch
    • Branch: main, /(root)

5단계: 코드 배포

생성된 사이트를 Github 저장소에 업로드합니다:

Shell
# 저장소 클론
$ git clone https://github.com/{username}/{username}.github.io.git
$ cd {username}.github.io

# 빌드된 파일 복사 (저장소의 기존 파일 삭제 후 my-gatsby-site/public 폴더 내용 복사)

# Github 업로드
$ git add .
$ git commit -m "Deploy Gatsby blog site"
$ git push -u origin main

6단계: 배포 완료 대기

코드 업로드 완료 후 https://{github username}.github.io로 접속하여 사이트를 확인할 수 있습니다.

주의사항: 첫 번째 배포의 경우 사이트 활성화까지 10-30분 정도 소요될 수 있습니다.

문제 해결 참고사항

프로젝트 진행 중 발생할 수 있는 주요 문제들과 해결 방법:
